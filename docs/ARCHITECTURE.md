# labrat: Architecture

## The Four Layers (v2)

```
┌─────────────────────────────────────────────┐
│  Layer 1: INFRASTRUCTURE (immutable)        │
│  Your eval pipeline: data -> model -> metrics│
│  The AI cannot modify this.                 │
├─────────────────────────────────────────────┤
│  Layer 2: EXPERIMENTS (AI-generated)        │
│  Config YAMLs: what to test this cycle      │
│  Single-delta from parent (multi-delta for  │
│  capstone). AI generates.                   │
├─────────────────────────────────────────────┤
│  Layer 3: CONTROL (the orchestrator)        │
│  orchestrator.md: how to behave             │
│  constitution.md: how to score              │
│  branches.yaml: what to explore             │
│  State files: what happened so far          │
├─────────────────────────────────────────────┤
│  Layer 4: EXTERNAL KNOWLEDGE (new in v2)    │
│  Research scouts: find papers and repos     │
│  Expansion agents: inject new ideas         │
│  Consolidation: distill findings            │
│  Cross-lab inheritance: reuse knowledge     │
└─────────────────────────────────────────────┘
```

### Layer 1: Infrastructure
Your existing code. The AI calls it but can't modify it. This prevents
the AI from gaming the evaluation metric.

### Layer 2: Experiments
Config files generated by the AI each cycle. Each config changes ONE parameter
from its parent (single-delta rule), except for capstone branches which support
multi-delta combinations. Three experiment types:
- **standard**: config -> run -> metrics -> judge -> verdict
- **diagnostic**: config -> run -> extract insights -> no composite score
- **meta**: re-evaluate previous experiments under new information

### Layer 3: Control
Markdown and YAML files that define the AI's behavior. The orchestrator is
the "agent brain." The constitution defines scoring rules. Branches define
the search space. State files track what happened.

### Layer 4: External Knowledge (new)
Agents that reach outside the lab for new ideas:
- **Tree designer**: surveys the landscape before the lab starts
- **Research scout**: searches for papers when a branch is stuck
- **Expansion scout**: injects orthogonal approaches at intervals
- **Consolidation agent**: distills findings into structured knowledge

## State Management

```
state/
├── cycle_counter.json       # Current cycle, total experiments, last transition
├── branch_beliefs.json      # Per-branch: EV, uncertainty, notes, experiment type
├── champions.json           # Best experiment per branch + production champion
├── budget.json              # Remaining compute credits per branch
├── experiment_log.jsonl     # Append-only history of all experiments
├── active_agents.json       # Live subagent status for dashboard
├── transition_log.jsonl     # Named cycle transition types (new)
├── belief_chain.json        # Assumption dependencies and invalidations (new)
├── scout_history.json       # External search tracking (new)
└── decay_tracking.json      # Val/test performance decay ratios (new)
```

**No database. No external service.** The AI reads JSON, makes decisions,
writes JSON. State is human-readable, diffable, and git-trackable.

## The Scoring Pipeline

```
Experiment Result
    │
    ▼
Hard Gates (binary pass/fail)
    │ metric > baseline?
    │ p-value < 0.10?
    │ model not collapsed?
    │
    ▼ (if all pass)
Soft Score (0-1 composite)
    │ D: deployment metric (0.35)
    │ R: robustness (0.20)
    │ G: generalization (0.15)
    │ I: information gain (0.10)  <- NEW: are predictions different?
    │ C: calibration (0.10)       <- FIXED: computed per domain
    │ K: complexity penalty (0.05)
    │
    ▼
Effect Size Gate (new)
    │ Cohen's d > 0.05? (not just noise)
    │
    ▼
Verdict
    │ PROMOTE: score >= champion AND d > 0.05
    │ MARGINAL: within 0.05 of champion
    │ REJECT: below champion - 0.05
    │ DIAGNOSTIC: no scoring (diagnostic experiments)
```

### Per-Branch Scoring (new)

Branches can override default weights in `branches.yaml`:
```yaml
execution:
  scoring:
    primary_metric: "phantom_sharpe_50ms"
    weights: {D: 0.50, R: 0.20, G: 0.15, I: 0.10, C: 0.00, K: -0.05}
```

### Val/Test Decay Tracking (new)

Every PROMOTE logs `decay_ratio = test_metric / val_metric`. If the rolling
average drops below 0.50, flag a methodology problem.

## The Allocator

UCB1-inspired branch selection balances exploration vs exploitation.
Shared logic in `scripts/allocator.py` (used by both orchestrator and batch_runner).

```
priority = 0.3 * EV + 0.4 * uncertainty + 0.3 * recency

EV = exponential moving average of improvement rate
uncertainty = 1 / sqrt(1 + n_experiments)  [decays with exploration]
recency = min(1, cycles_since_last / 3)    [rewards waiting]
```

### Diminishing Returns Detection (new)

Track `composite_score_delta` across ALL branches. If the last 8 experiments
produce delta < 0.005, the lab is effectively converged -- even if individual
branches haven't hit the stuck threshold.

### Stuck Branch Threshold

Raised from 3 to 4 consecutive non-improvements (too aggressive at 3 for
random search in noisy domains). Configurable in branches.yaml via
`budget_rules.exhaustion_threshold`.

## The Loop (v2)

```
Human: "Read orchestrator.md and execute one cycle"
  │
  ▼
GRADUATED READING (new)
  │ Level 0: handoff.md + cycle_counter.json
  │ Level 1: + branch_beliefs + budget (if allocating)
  │ Level 2: + champions + dead_ends (if scoring)
  │ Level 3: + experiment_log + branches.yaml (if stuck/scouting)
  │
  ▼
EXPANSION CHECK (every 20th cycle, new)
  │ Deploy expansion scout instead of running experiments
  │ Scout uses WebSearch to find external approaches
  │
  ▼
RESEARCH SCOUT CHECK (new)
  │ Any stuck branches? Deploy research scout
  │
  ▼
AI picks branches via allocator
  │
  ▼
AI generates experiment configs
  │
  ▼
PARALLEL EXECUTION via subagents
  │
  ▼
Judge scores mechanically
  │
  ▼
SYNTHESIS STEP (new)
  │ "What did we learn? Should we change direction?"
  │ The orchestrator MUST understand findings, not just relay
  │
  ▼
AI updates state files (sequential)
  │
  ▼
BELIEF REVISION CHECK (new)
  │ Any findings that invalidate previous experiments?
  │
  ▼
TRANSITION TYPE (new)
  │ experiments_complete | convergence | surprise_detected |
  │ frame_invalidation | human_checkpoint | new_hypothesis
  │
  ▼
AI writes handoff
  │
  ▼
/loop fires again
```

## Parallel Branch Execution

Branches are independent. The orchestrator runs experiments across branches
concurrently via subagents.

### The Parallel Model

```
Orchestrator reads state
    │
    ├── Select N branches by priority (shared allocator.py)
    ├── Generate N configs (one per branch)
    ├── Write active_agents.json (dashboard shows pulsing dots)
    │
    ├── Launch N subagents IN PARALLEL ─────────────────────────┐
    │     ├── Agent: features branch -> run_experiment + judge  │
    │     ├── Agent: model branch -> run_experiment + judge      │ concurrent
    │     ├── Agent: objectives branch -> run_experiment + judge │
    │     └── Agent: preprocessing branch -> run_experiment + judge
    │                                                            │
    ├── Collect all results <────────────────────────────────────┘
    │
    ├── SYNTHESIS: What did we learn? (new)
    ├── Update state SEQUENTIALLY (shared files)
    ├── Check for belief revision triggers (new)
    ├── Clear active_agents.json
    └── Write handoff with transition type

```

### Subagent Resilience (new)

- Per-subagent timeouts (configurable per branch)
- Partial cycle completion: process finished agents, don't block on stragglers
- Crash recovery: check for orphaned active_agents.json entries on startup

### What Parallelizes and What Doesn't

| Step | Parallel? | Why |
|------|-----------|-----|
| Read state | No | Single read at cycle start |
| Select branches | No | Depends on state |
| Generate configs | Yes | Independent per branch |
| Run experiments | **Yes** | Fully independent |
| Run judge | **Yes** | Independent per experiment |
| Synthesis | No | Needs all results |
| Update state | **No** | Shared files |
| Belief revision | No | Depends on state |
| Write handoff | No | Single summary |

## External Knowledge Agents (new)

### Tree Designer (Phase 0)

Before the lab starts, the tree designer surveys the landscape:
1. Takes mission statement + baseline config
2. Uses WebSearch to find state-of-the-art approaches
3. Designs 5-8 branches with concrete search_space entries
4. Writes branches.yaml, dead_ends.md, research_brief.md

Prompt: `labrat/templates/tree_designer.md`

### Research Scout (Phase 1, on stuck)

When a branch has 4+ consecutive non-improvements:
1. Reviews branch experiment history + dead ends
2. Uses WebSearch for papers, repos, and techniques
3. Proposes 3-5 new experiment configs as YAML
4. Orchestrator reviews and adds to branches.yaml

Prompt: `labrat/templates/research_scout.md`
Script: `labrat/scripts/research_scout.py`

### Expansion Scout (Phase 2, every 20 cycles)

Periodic injection of external knowledge:
1. Reviews current findings and dead ends
2. Searches for approaches the lab HASN'T tried
3. Proposes 2-3 new branches with full search_space
4. Prevents local optima by finding orthogonal directions

Prompt: `labrat/templates/expansion_scout.md`

### Consolidation Agent (Phase 3, every 10 cycles)

Distills experiment history into structured knowledge:
1. Groups experiments by branch, identifies patterns
2. Updates FINDINGS.md with scaling curves and dead ends
3. Proposes branch_beliefs and dead_ends updates
4. Writes human-readable research digest

Prompt: `labrat/templates/consolidation_agent.md`

## Experiment Types (new)

| Type | Scoring | Example |
|------|---------|---------|
| standard | composite score, verdict | Change learning rate, measure F1 |
| diagnostic | insights only, verdict=DIAGNOSTIC | Delay audit, data leakage check |
| meta | can change previous verdicts | Re-score experiments with corrected metrics |

Diagnostic experiments are logged in experiment_log.jsonl but don't affect
branch_beliefs or champions. They inform the synthesis step.

Meta experiments can trigger belief revision: retroactively invalidating
previous experiments by adding `invalidated_by` fields.

## Belief Revision (new)

Tracked in `state/belief_chain.json`:

```json
{
  "assumptions": {
    "instant_fill": {
      "stated_cycle": 0,
      "description": "Assume fills at signal time (0ms delay)",
      "depends_on": [],
      "experiments_using": ["exec_phantom_c1", "exec_phantom_c3"]
    }
  },
  "invalidations": [
    {
      "cycle": 9,
      "assumption": "instant_fill",
      "finding": "Phantom delay has no sub-second resolution",
      "affected_experiments": ["exec_phantom_c1", "exec_phantom_c3"],
      "action": "Re-score with corrected delay estimates"
    }
  ]
}
```

When a meta experiment invalidates an assumption, the orchestrator:
1. Logs the invalidation
2. Flags affected experiments in experiment_log.jsonl
3. Triggers a `frame_invalidation` transition
4. Pauses for human review (or auto-re-scores if configured)

## Cross-Lab Knowledge Transfer (new)

### Lab Registry

`lab_registry.json` at the project root tracks all labs:

```json
{
  "labs": {
    "btc_v1": {"path": "...", "mission": "...", "status": "converged"},
    "btc_v2": {"path": "...", "mission": "...", "status": "converged"},
    "prediction_markets": {"path": "...", "mission": "...", "status": "active"}
  }
}
```

### Inheritance

```bash
python scripts/bootstrap.py --inherit-from /path/to/parent/lab
```

Copies dead_ends.md, FINDINGS.md, and champions.json from the parent lab.

## Transition Types (new)

Every cycle ends with a named transition logged in `state/transition_log.jsonl`:

| Transition | Meaning |
|-----------|---------|
| experiments_complete | Normal cycle end |
| budget_exhausted | All branches at 0 |
| convergence | No branch can improve |
| surprise_detected | Result >3 sigma from expected |
| frame_invalidation | Finding changes scoring rubric |
| human_checkpoint | Every 15 cycles, pause for review |
| stuck_all_branches | Every active branch stuck |
| new_hypothesis_proposed | Agent wants to add a branch |
| expansion_complete | Expansion scout finished |
| consolidation_complete | Consolidation agent finished |

## Hooking Into Different Systems

### Claude Code
Native `/loop` command + `Agent` tool for parallel branches. The primary target.
All external knowledge agents (scout, expansion, consolidation) run as subagents.

### Headless (cron + API)
```python
import anthropic
client = anthropic.Anthropic()
orchestrator = open("research_lab/orchestrator.md").read()
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=16000,
    system=orchestrator,
    messages=[{"role": "user", "content": "Execute one research cycle."}],
    tools=[...],
)
```

### GitHub Actions
```yaml
name: Research Cycle
on:
  schedule:
    - cron: '0 * * * *'
jobs:
  cycle:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: python run_one_cycle.py
      - run: git add research_lab/state/ research_lab/logs/
      - run: git commit -m "Lab cycle $(cat research_lab/state/cycle_counter.json | jq .cycle)"
      - run: git push
```
