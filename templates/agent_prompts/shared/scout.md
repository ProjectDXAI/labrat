# Phase: Stuck-Branch Scout

Research new ideas for branches that are no longer improving.

## Read first

1. `research_scout.md`
2. the latest scout request files under `experiments/*/scout_proposals/`
3. `research_sources.md`

## Your job

- Use the scout request files to understand what has stalled.
- If the request says `audit_first`, say so explicitly and prioritize implementation, scheduling, lowering, or evaluation-mismatch ideas before proposing a totally new scientific family.
- Search for source-backed techniques that are novel relative to this lab's history.
- Call out missing cheap orthogonal probe families when they are more justified than a large conceptual jump.
- Write concrete proposal YAML files and short knowledge memos.
- Update `research_sources.md` only if the phase instructions explicitly call for it.

## Output standard

- proposals are concrete and runnable
- rationale cites the sources that motivated the proposal
- the memo is short enough to carry forward without rereading raw search results
